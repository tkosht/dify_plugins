# Standards Sources

Use these primary references when mapping findings and writing remediation rationale.

## Core Standards
- OWASP ASVS (latest stable release stream)  
  https://github.com/OWASP/ASVS
- OWASP API Security Top 10 (2023)  
  https://owasp.org/API-Security/editions/2023/en/0x11-t10/
- OWASP Top 10:2021  
  https://owasp.org/Top10/2021/
- MITRE CWE Top 25 (2024)  
  https://cwe.mitre.org/top25/archive/2024/2024_top25_list
- NIST SP 800-218 SSDF v1.1  
  https://csrc.nist.gov/pubs/sp/800/218/final

## Tooling References
- Bandit  
  https://github.com/PyCQA/bandit
- pip-audit  
  https://pypi.org/project/pip-audit/

## Usage Rules
- Prefer primary sources over blog summaries.
- Use domain/chapter mapping if exact control IDs are uncertain.
- Do not invent IDs or claim compliance not supported by evidence.
